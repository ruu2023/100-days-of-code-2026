# Workflow: Save Technical Explanation

## Goal
現在のチャットセッションで行われたコード解説や技術的な議論を、後で参照しやすいようにMarkdownドキュメントとして抽出・保存する。

## Steps
1. **履歴の解析**: 
   - これまでの会話の中から、解説された主要な「コード片」と「技術的なポイント」を特定する。
2. **ドキュメント生成**:
   - `docs/explanations/` ディレクトリ配下に、今日の日付とトピックを含んだファイル名（例: `2026-03-03-hono-middleware-logic.md`）で新規作成する。
3. **内容の構成**:
   - 冒頭に「何の解説か」の概要を書く。
   - 修正前・修正後のコードをコードブロックで記述する。
   - 重要な注意点や、今後の開発で活かせるポイントを箇条書きでまとめる。
4. **完了報告**:
   - 保存したファイルパスを表示して終了する。
